# Sample Data Files

This directory contains sample/example output files demonstrating the structure and content of analysis and insights output.

## Files

- **sample_analysis.json** - Example Phase 1 analysis output
  - Shows real analysis from a Kubernetes cluster with Prometheus metrics
  - Demonstrates all analysis sections: deployments, HPAs, nodes, cross-layer observations
  - Can be used for UI development/testing without running live analysis

- **sample_insights.json** - Example Phase 2 insights output
  - Shows LLM-generated insights based on Phase 1 analysis
  - Demonstrates insights structure: patterns, warnings, action candidates
  - Shows confidence levels and scope for each insight

## Usage

### For Development
Use sample files to test the UI without needing a live Prometheus instance:

```bash
cp examples/sample_analysis.json output/analysis_output.json
cp examples/sample_insights.json output/insights_output.json
python ui.py
```

Then visit http://127.0.0.1:8080

### For Testing
Load these in tests to verify UI rendering and data handling:

```python
import json
with open('examples/sample_analysis.json') as f:
    analysis = json.load(f)
with open('examples/sample_insights.json') as f:
    insights = json.load(f)
```

## Data Characteristics

### sample_analysis.json
- **Deployments**: 9 production deployments with real resource metrics
- **HPAs**: 4 autoscaling configurations with scaling behavior data
- **Nodes**: Multi-node cluster with CPU/memory allocation
- **Scope**: 30-minute observation window with sufficient data for analysis

### sample_insights.json
- **Generated by**: Ollama llama3:8b model
- **Patterns**: Detected memory over-provisioning, CPU burst behavior
- **Warnings**: 2-3 medium-to-high confidence warnings
- **Actions**: 2-3 candidate areas for review
- **Confidence levels**: Ranges from 0.6 to 0.95

## Updating Samples

When the analysis or insights format changes, update samples by running:

```bash
python orchestrator.py
PHASE2_ENABLED=true python orchestrator.py
cp output/analysis_output.json examples/sample_analysis.json
cp output/insights_output.json examples/sample_insights.json
```

Then commit the new samples to version control.
